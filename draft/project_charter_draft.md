## บทนำ

จักรยานเป็นพาหนะที่ไม่ก่อมลพิษ มีความคล่องตัว และสามารถใช้งานได้หลากหลายทั้งการเดินทางในชีวิตประจำวัน การออกกำลังกาย และการฝึกซ้อมเพื่อการแข่งขัน ผู้ใช้แต่ละกลุ่มจึงมีความต้องการข้อมูลที่แตกต่างกัน โดยเฉพาะกลุ่มที่ต้องการติดตามสมรรถนะการปั่นมักให้ความสำคัญกับตัวชี้วัด เช่น ระยะทาง เวลา ความเร็ว อัตราการเต้นหัวใจ และกำลังปั่น

ในบรรดาตัวชี้วัดดังกล่าว กำลังปั่นเป็นค่าที่สะท้อนอัตราการส่งพลังงานจากผู้ปั่นผ่านขาจานเข้าสู่ระบบขับเคลื่อนของจักรยานได้โดยตรง จึงมีประโยชน์ต่อการควบคุมการออกแรงและการวางแผนการซ้อม อย่างไรก็ตาม เครื่องวัดกำลังปั่นที่มีจำหน่ายในตลาดยังมีราคาค่อนข้างสูง และบางรุ่นมีข้อจำกัดด้านความเข้ากันได้กับชิ้นส่วนเดิมของจักรยาน โครงงานนี้จึงเสนอการพัฒนาระบบต้นแบบแบบเสริมที่ติดตั้งบนขาจานข้างเดียว โดยใช้ `strain gauge` และ `IMU` ร่วมกับ `ESP32` เพื่อคำนวณกำลังปั่นเบื้องต้น ส่งข้อมูลผ่าน `BLE` ไปยังแอปพลิเคชันบนระบบ `Android` และจัดเก็บข้อมูลลง `Firebase Cloud Firestore`

## ขอบเขตของโครงการ

- พัฒนาต้นแบบระบบตรวจวัดกำลังปั่นแบบ `single crank arm based system`
- ติดตั้ง `strain gauge` บน crank arm เดิมเพื่อวัดการเปลี่ยนรูปของวัสดุ
- ใช้ `HX711` หรือวงจรอ่านสัญญาณเทียบเท่าในการอ่านค่าสัญญาณจาก `strain gauge`
- ใช้ `IMU` รุ่น `MPU6050` เพื่อวัดความเร็วเชิงมุมของขาจาน
- ใช้ `ESP32` เป็นหน่วยประมวลผลบนตัวจักรยาน และใช้พลังงานจากแบตเตอรี่ `7.4V`
- ใช้โครงจักรยานพับ รุ่น `Fika Frappe` เป็นแพลตฟอร์มทดลองของต้นแบบ
- ใช้แอปพลิเคชันบนระบบ `Android` รับข้อมูลผ่าน `BLE` แสดงผล ติดตามเส้นทาง และจัดเก็บข้อมูลการปั่น
- ใช้ `Google Maps Flutter SDK` และ `Geolocator` สำหรับการแสดงแผนที่และการติดตามเส้นทาง
- ส่งข้อมูลจากมือถือขึ้น `Firebase Cloud Firestore` ผ่านเครือข่ายมือถือ `4G / 5G`
- ทดสอบความเสถียรของการอ่านสัญญาณ การสื่อสารผ่าน `BLE` การติดตามเส้นทาง และการบันทึกข้อมูลบนคลาวด์

## วัตถุประสงค์ของโครงการ

1. เพื่อพัฒนาต้นแบบระบบที่สามารถอ่านค่ากำลังปั่นจาก crank arm แบบเสริมได้โดยไม่ต้องเปลี่ยนชิ้นส่วนหลักของจักรยาน
2. เพื่อพัฒนาให้ `ESP32` สามารถรับข้อมูลจาก `strain gauge` และ `IMU` แล้วคำนวณกำลังปั่นเบื้องต้นบนตัวอุปกรณ์
3. เพื่อพัฒนาแอปพลิเคชันบนระบบ `Android` สำหรับรับข้อมูลผ่าน `BLE` และแสดงผลข้อมูลการปั่นแบบเรียลไทม์
4. เพื่อพัฒนาแอปพลิเคชันให้สามารถติดตามเส้นทางและบันทึกข้อมูลการปั่นขึ้นฐานข้อมูลบนคลาวด์ได้
5. เพื่อศึกษาความเป็นไปได้เชิงเทคนิค ข้อจำกัด และแนวทางต่อยอดของระบบวัดกำลังปั่นต้นทุนต่ำ

## ประโยชน์ที่คาดว่าจะได้รับ

1. ได้ต้นแบบระบบ IoT สำหรับตรวจวัดกำลังปั่นจักรยานที่สามารถต่อยอดได้ในอนาคต
2. ได้ประสบการณ์ในการบูรณาการฮาร์ดแวร์ วงจรไฟฟ้า เฟิร์มแวร์ แอปพลิเคชันบนมือถือ และฐานข้อมูลบนคลาวด์เข้าด้วยกัน
3. ได้ข้อมูลการปั่นและเส้นทางที่สามารถนำไปใช้วิเคราะห์พฤติกรรมการปั่นในภายหลังได้
4. ได้แนวทางศึกษาหลักการของเครื่องวัดกำลังปั่นในต้นทุนที่ต่ำกว่าการใช้อุปกรณ์เชิงพาณิชย์
5. สามารถใช้เป็นพื้นฐานสำหรับการพัฒนาระบบวิเคราะห์สมรรถนะการปั่นหรือผลิตภัณฑ์ที่เกี่ยวข้องต่อไป
